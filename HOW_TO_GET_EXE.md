# راهنمای گرفتن فایل EXE / Installer برای NovaSort Pro با GitHub Actions

## چرا خودم نمی‌تونم اینجا exe بسازم؟
NovaSort Pro یک برنامه‌ی **WinUI 3 / .NET 9** است که فقط روی ویندوز (با Windows SDK) کامپایل می‌شود.
محیط من یک سندباکس **لینوکسی** بدون دسترسی به اینترنت است، پس نه dotnet ویندوزی دارم و نه امکان کامپایل WinUI 3 را دارم.

## راه‌حل: ساخت خودکار روی سرورهای گیت‌هاب (رایگان)
چون سیستم خودت ضعیفه، بهترین راه اینه که کامپایل رو به یک "رانر ویندوزی" در GitHub Actions بسپاری (کاملاً رایگان برای پروژه‌های عمومی/خصوصی کوچک). دو فایل زیر رو برات آماده کردم:

1. `.github/workflows/build.yml` → یک ورک‌فلو که:
   - روی یک ماشین ویندوزی در فضای ابری اجرا می‌شود
   - پروژه رو با `dotnet publish` به‌صورت **self-contained + single-file** کامپایل می‌کند → یک `NovaSortPro.exe` قابل حمل تولید می‌شود
   - با **Inno Setup** یک **installer واقعی** (`NovaSortPro-Setup-1.0.0.exe`) هم می‌سازد
   - هر دو فایل رو به‌عنوان Artifact قابل دانلود آپلود می‌کند

2. `installer/setup.iss` → اسکریپت Inno Setup که installer رسمی رو می‌سازد.

## مراحل استفاده
1. یک ریپازیتوری جدید در GitHub بساز (خصوصی یا عمومی، فرقی نداره).
2. کل پوشه `NovaSortPro` (سورس اصلی) + دو فایل بالا (`.github/workflows/build.yml` و `installer/setup.iss`) رو داخل ریشه ریپو قرار بده. ساختار نهایی باید این‌طوری باشه:
   ```
   your-repo/
   ├── NovaSortPro/          ← همون پوشه سورس که آپلود کرده بودی
   ├── installer/
   │   └── setup.iss
   └── .github/
       └── workflows/
           └── build.yml
   ```
3. تغییرات رو push کن (به شاخه `main`).
4. برو به تب **Actions** توی ریپو → ورک‌فلو "Build NovaSort Pro" به‌صورت خودکار اجرا می‌شود (چند دقیقه طول می‌کشد).
5. بعد از اتمام، پایین صفحه‌ی همون اجرا (run) دو Artifact می‌بینی:
   - `NovaSortPro-portable-exe` → فقط exe قابل حمل
   - `NovaSortPro-installer` → installer کامل (Setup.exe)

   هر دو رو می‌تونی دانلود کنی.

### اگه می‌خوای هر بار خودکار Release هم بسازه
یک تگ نسخه بزن، مثلاً:
```bash
git tag v1.0.0
git push origin v1.0.0
```
این کار باعث می‌شود ورک‌فلو یک **GitHub Release** با فایل‌های exe/installer پیوست شده بسازد — لینک دانلود مستقیم و همیشگی.

## نکته درباره امضای دیجیتال (Code Signing)
چون exe امضا نشده، ویندوز و مرورگرها هنگام اجرا هشدار "ناشناس" نشان می‌دهند. این طبیعی است برای پروژه‌های شخصی بدون گواهی Authenticode؛ در README خودت هم اشاره شده بود که ساختار پروژه آماده امضاست، اما امضا کردن نیاز به گواهی پولی دارد و اختیاری است.
